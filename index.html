ESE Companion v2

Centralized management for HiveMQ Enterprise Security Extension databases. Manage multiple databases from a single interface.

PostgreSQL MySQL SQL Server

0 Prerequisites

  • Docker and Docker Compose installed
  • Git installed
  • Ports 5432, 5433, 3306, 1433, 8989, and 9090 available

1 Clone the Repository

git clone https://github.com/anthonyolazabal/ese-companion.git
cd ese-companion

2 Start the Databases

The included docker-compose.yaml starts four database containers:

  • companion-db — PostgreSQL 16 for ESE Companion app data (port 5432)
  • ese-postgresql — PostgreSQL 16 as a sample ESE database (port 5433)
  • ese-mysql — MySQL 8.4 as a sample ESE database (port 3306)
  • ese-mssql — Azure SQL Edge as a sample ESE database (port 1433)
docker compose up -d

Wait for all containers to be healthy:

docker compose ps

3 Start ESE Companion

Option A: Docker Image (Recommended)

docker run -d --name ese-companion \
  --network host \
  -e ESE_COMPANION_DB_TYPE=postgresql \
  -e ESE_COMPANION_DB_HOST=localhost \
  -e ESE_COMPANION_DB_PORT=5432 \
  -e ESE_COMPANION_DB_NAME=companion \
  -e ESE_COMPANION_DB_USER=companion \
  -e ESE_COMPANION_DB_PASSWORD=companion \
  -e ESE_COMPANION_JWT_SECRET=change-me-min-16-chars \
  -e ESE_COMPANION_ENCRYPTION_KEY=change-me-min-16-chars \
  -e ESE_COMPANION_ADMIN_USER=admin \
  -e ESE_COMPANION_ADMIN_PASSWORD=admin \
  -e ESE_COMPANION_ADMIN_EMAIL=admin@example.com \
  anthonyolazabal/ese-companion:latest
Access the application:
HTTP: http://localhost:8989  |  HTTPS: https://localhost:9090 (self-signed certificate)

6 Browse ESE Entities

Click on a connection to see its ESE entities. Three domain tabs are available:

  • MQTT — Users, roles, and topic-based permissions
  • Control Center — Users, roles, and 49 pre-loaded HiveMQ CC permissions
  • REST API — Users, roles, and 47 pre-loaded HiveMQ REST API permissions

7 Next Steps

  • Create ESE users — 6 hash algorithm options (PLAIN, MD5, SHA512, PKCS5S2, BCrypt, Argon2id)
  • Set up roles & permissions — define roles, assign permissions, then assign roles to users
  • Manage companion users — Admin > Users for additional users with admin/readwrite/readonly roles
  • Generate API keys — Settings > API Keys for scoped, expiring programmatic access
  • Explore the API — /api/v1/docs for interactive Swagger UI
  • Review audit logs — Admin > Audit Logs for a record of all actions

Stopping Everything

# Stop ESE Companion (if running via Docker)
docker stop ese-companion && docker rm ese-companion

# Stop the databases
docker compose down

# Remove all data
docker compose down -v
